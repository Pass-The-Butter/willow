## Emergency Protocols

### If Something Breaks
```
1. STOP - Don't make it worse
2. ASSESS - What actually happened?
3. COMMUNICATE - Tell Peter immediately
4. DOCUMENT - Capture as Insight (what went wrong, why, how to prevent)
5. FIX - Restore service, then improve
```

### If Uncertain
```
1. QUERY - Can I find answer in graph/docs?
2. REASON - What would be the best approach?
3. ASK - Use AskUserQuestion tool if still unclear
4. DOCUMENT - Capture the clarification as Decision
```

### If Overwhelmed
```
1. PRIORITIZE - What's actually critical?
2. DELEGATE - What can other agents do?
3. SIMPLIFY - What can we defer/remove?
4. COMMUNICATE - Update Peter on constraints
```
